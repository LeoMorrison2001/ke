# 前端 UI 布局与配色规范

本规范用于小可桌面端的所有前端页面。新增或调整界面时，优先复用现有主题变量与通用样式；不要在单个组件中新增一套独立的浅色/深色配色。

## 1. 主题与配色

主题由 `html[data-theme='light' | 'dark']` 控制。颜色语义变量定义在 `src/renderer/src/App.vue`，组件应使用变量而非直接写十六进制颜色。

| 用途 | 变量 | 浅色主题 | 深色主题 |
| --- | --- | --- | --- |
| 窗口标题栏背景 | `--color-titlebar` | `#f5f5f5` | `#141414` |
| 页面与工具栏背景 | `--color-page` / `--color-surface` | 白色 | `#181818` |
| 卡片、菜单、弹窗背景 | `--color-surface` | 白色 | `#181818` |
| 表头、弱表面 | `--color-surface-muted` | 浅灰 | `#202020` |
| 悬停表面 | `--color-surface-hover` | 浅灰 | `#303030` |
| 主文字 | `--color-text` | 深灰 | 浅灰白 |
| 次级文字 | `--color-text-muted` | 中灰 | 浅灰 |
| 弱文字、禁用态 | `--color-text-subtle` | 浅灰 | 深灰 |
| 边框与分隔线 | `--color-border` / `--color-border-subtle` | `#e3e6ea` | `#242424` |
| 主强调色 | `--color-accent` | 绿色 | 深绿色 |
| 强调色悬停 | `--color-accent-hover` | 深绿色 | 深绿色 |
| 强调文字 | `--color-accent-text` | 绿色 | 浅绿色 |
| 弱强调背景 | `--color-accent-soft` | 浅绿色 | 深绿灰 |
| 危险操作 | `--color-danger` | 红色 | 红色 |

### 强调色规则

- 全局唯一主强调色是绿色。
- 深色主题中，窗口顶部标题栏使用 `#141414`；页面、工具栏、卡片、侧栏和弹窗使用 `#181818`。标题栏不显示底边框。
- 主按钮、确认操作、发送按钮、选中菜单、成功状态、可点击的主操作使用绿色语义变量。
- 不要为常规操作引入蓝色按钮、蓝色消息气泡或蓝色链接状态。
- 红色只用于删除、错误与不可恢复的危险操作。
- 棕色等颜色仅可作为图标本身的装饰色，不能作为交互强调色。

## 2. 页面布局

### 窗口层级

窗口顶部标题栏高度为 `32px`，应用内容从标题栏下方开始。固定定位的内容层（例如模态框）不得遮罩标题栏。

首页历史抽屉背景必须复用 `--color-titlebar`，与窗口顶部标题栏保持相同颜色。

### 一级页面

首页、应用、设置均为一级页面：

- 使用 `.page-view` 或同等的全高页面容器。
- 顶部工具栏高度为 `48px`，不显示底边框。
- 主内容区默认内边距使用 `var(--content-padding)`。
- 窄屏内容区内边距使用 `var(--content-padding-compact)`。

### 应用二级页面

小可记忆、小可日记等从应用页进入的页面：

- 顶部工具栏同样无底边框。
- 左侧菜单宽度为 `184px`；窄屏为 `132px`。
- 右侧内容区以左边框和左上圆角与菜单区分，不额外给工具栏添加边线。
- 菜单默认、悬停、选中态分别使用正常表面、悬停表面和弱强调背景。

## 3. 通用组件规则

### 按钮

- 主按钮：`--color-accent` 背景、`--color-on-accent` 文字；悬停使用 `--color-accent-hover`。
- 次按钮/轮廓按钮：表面背景、标准边框、次级文字；悬停使用 `--color-surface-hover`。
- 危险按钮：`--color-danger`。
- 禁用态降低对比度，且不可使用主强调色表示可操作。

### 表格

- 使用标准边框 `--color-border`，单元格分隔使用 `--color-border-subtle`。
- 表头背景使用 `--color-surface-muted`，文字使用 `--color-text-muted`。
- 表格内容默认使用主/次级文字变量；可操作文字使用 `--color-accent-text`。
- 表格在窄屏保持最小宽度并横向滚动，不压缩到不可读。

### 模态框

- 所有模态框必须使用 `.modal-backdrop` 与 `.modal-panel`。
- `.modal-backdrop` 统一从标题栏下方开始覆盖：`inset: 32px 0 0`。
- 遮罩使用 `--modal-backdrop-color`，不得各组件自行指定颜色、范围或层级。
- `.modal-panel` 统一提供表面背景、主文字、`14px` 圆角和 `--shadow-modal` 阴影。
- 弹窗尺寸、内部内容布局可以按场景变化；标题、说明、输入框和底部操作区保持相同的主题语义。

## 4. 响应式规则

统一使用 `640px` 作为窄屏断点：

- 页面内容内边距切换为 `--content-padding-compact`。
- 二级页面侧栏宽度切换为 `132px`。
- 设置页的操作区可改为纵向排列。
- 首页聊天消息区与输入框在窄屏宽度为 `100%`。
- 模态框宽度必须受视口限制，避免水平溢出。

不要为普通页面新增不同的 `560px`、`600px` 等断点；如确有特殊需求，应在代码旁说明原因。

## 5. 开发检查清单

新增 UI 前后，请确认：

- 浅色和深色主题都只使用主题变量或已有通用样式。
- 主交互是否保持绿色，不存在临时蓝色强调。
- 一级/二级页面是否符合对应的工具栏与菜单布局规则。
- 模态框是否复用统一遮罩和面板类，且未覆盖窗口标题栏。
- `640px` 窄屏下内容、表格、菜单和模态框均可使用。
- 已执行 `npm run typecheck`、`npm run lint`；涉及样式时还应人工检查浅色与深色主题。
